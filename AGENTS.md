# AI Preferences

## Git Commit Messages

## Code Modification Principles

### 区分通用问题与例外处理
- 问题分析优先于代码修改
- 通用问题用通用方案解决，不要用特例 workaround
- 例外情况特殊处理，不累积到通用逻辑中
- 避免"检测-删除"式的事后补救，优先从源头避免问题

### 代码修改统筹考虑
- 每次修改要考虑对整体架构的影响
- 不要为了修复一个问题而破坏已有的成熟逻辑
- 如果方案会增加代码复杂度，说明方向可能不对，考虑更优雅的方案
